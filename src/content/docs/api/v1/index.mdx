This page provides an overview of available API endpoints in Akari's platform. All endpoints use the base URL: `https://akarimanga.dpdns.org/api/v1`.

  <TabContent id="manga">
    ## [Manga API](./latest/manga)

    Access manga content, search for titles, and browse collections.

    | Endpoint | Description |
    |----------|-------------|
    | `GET /manga/[id]` | Get manga details by ID |
    | `GET /manga/[id]/[subId]` | Get specific chapter/volume |
    | `GET /manga/list/latest` | List recently updated manga |
    | `GET /manga/list/popular` | List popular manga |
    | `GET /author/[id]` | List manga by author |
    | `GET /author/[id]/og` | Get author open graph image |
    | `GET /genre/[id]` | Browse manga by genre |
    | `GET /search` | Search for manga titles |

  </TabContent>

  <TabContent id="account">
    ## Account API

    Manage user accounts and bookmarks.

    | Endpoint | Method | Description |
    |----------|--------|-------------|
    | `/bookmarks` | GET | List all bookmarks |
    | `/bookmarks/add` | POST | Add a bookmark |
    | `/bookmarks/all` | GET | Get all bookmarks |
    | `/bookmarks/batch-update` | PUT | Update multiple bookmarks |
    | `/bookmarks/bookmarked` | GET | Check if bookmarked |
    | `/bookmarks/delete` | DELETE | Remove a bookmark |
    | `/bookmarks/last-read` | GET | Get last read bookmark |
    | `/bookmarks/notification` | GET | Get bookmark notifications |
    | `/bookmarks/search` | GET | Search bookmarks |
    | `/bookmarks/update` | PUT | Update a bookmark |
    | `/login` | GET/POST | Authenticate user |
    | `/logout` | GET | End user session |
    | `/register` | GET/POST | Create new account |
    | `/captcha` | GET | Get captcha |

  </TabContent>

  <TabContent id="mal">
    ## MyAnimeList API

    Integrate with MyAnimeList for tracking and synchronization.

    | Endpoint | Method | Description |
    |----------|--------|-------------|
    | `/mal/isLoggedIn` | GET | Check MAL login status |
    | `/mal/logout` | GET | Logout from MAL |
    | `/mal/me` | GET | Get MAL user info |
    | `/mal/me/mangalist` | GET | Get user's MAL list |
    | `/mal/sync` | GET | Sync with MAL |
    | `/mal/token` | GET | Get MAL token |
    | `/mal/token/refresh` | GET | Refresh MAL token |

  </TabContent>
</Tabs>

## Response Format

API responses follow a consistent structure:

```typescript
{
    result: "success" | "error";
    status: number;
    data: any | { message: string; details?: unknown };
    performance?: {
        [key: string]: number;
    }
}
```

For more detailed documentation on specific endpoints, please refer to the individual API reference pages.
